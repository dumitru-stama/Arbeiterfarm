╔═════════════════════════════════════════════════════════════════════╗
║  SIGNAL PROTOCOL (agent output --> orchestrator)                    ║
║                                                                     ║
║  Agents emit signals as text in their response:                     ║
║                                                                     ║
║    signal:request:decompiler:found packed code                      ║
║    ───────┬───────────┬──────────────┬──────                        ║
║           │           │              └── reason (informational)      ║
║           │           └── target agent name                         ║
║           └── kind: request | skip | priority                       ║
║                                                                     ║
║  ┌───────────┬─────────────────────────────────────────────┐        ║
║  │  Kind     │  Effect                                     │        ║
║  ├───────────┼─────────────────────────────────────────────┤        ║
║  │ request   │  Add agent to next group (if not planned)   │        ║
║  │ skip      │  Remove agent from all remaining groups     │        ║
║  │ priority  │  Move agent from later group to next group  │        ║
║  └───────────┴─────────────────────────────────────────────┘        ║
║                                                                     ║
║  Rules:                                                             ║
║    * First signal per (target, kind) wins                           ║
║    * Request beats skip for same target                             ║
║    * Agents can only signal known workflow agents                   ║
║                                                                     ║

║  BUDGETS & LIMITS                                                   ║
║                                                                     ║
║  MAX_TOOL_CALLS ............. 20 per agent run                      ║
║  tool_call_budget (thinker).. 30 per thinker run                    ║
║  max_calls_per_run .......... 10 per tool per run                   ║
║  MAX_FANOUT_DEPTH ........... 1  (no nested fan-out)                ║
║  MAX_FANOUT_CHILDREN ........ 50 per parent artifact                ║
║  MAX_REPIVOTS ............... 5  per workflow execution             ║
║  max_input_bytes ............ 256 KB per tool input                 ║
║  max_input_depth ............ 16 JSON nesting levels                ║
║  max_output_bytes ........... 64 MB (redirected to artifact)        ║
║  tool timeout_ms ............ 60s default per tool                  ║
║  worker heartbeat ........... 30s (lease: 120s)                     ║
║  concurrent streams ......... per-user limit via stream_tracker     ║
║  daily LLM tokens ........... per-user quota                        ║
║  thinker timeout ............ 1800s (30 min)                        ║
║  child agent timeout ........ 300s  (5 min, < meta-tool timeout)    ║
║  meta-tool timeout .......... 660s  (11 min, > child timeout)       ║
║  compaction threshold ....... 0.85  (85% of context_window)         ║
║  max_read_artifact_bytes .... 128 KB per meta.read_artifact call    ║
